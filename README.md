# Hospital Management Database System

##  Project Overview
This project involves the design and implementation of a comprehensive database for a **Hospital Management System**. The goal was to create a structured environment to manage patients, doctors, appointments, medical records, and billing efficiently.

## Database Design
The design process followed standard database normalization forms and includes:

### 1. Entity-Relationship Diagram (ERD)
The ERD represents the logical structure of the hospital database, illustrating the relationships between patients, doctors, and medical services.
![Hospital ER Diagram](/design/ER_Diagram.jpg)

### 2. Relational Schema
This schema shows the final table structures, primary keys, and foreign key connections after normalization.
![Relational Schema](/design/Relational_Schema.jpg)

##  Key Features & Queries
The project includes a wide range of **SQL Queries** to handle real-world scenarios:
- **Data Definition (DDL):** Creating tables with specific constraints (Primary Keys, Foreign Keys, Not Null).
- **Data Manipulation (DML):** - Advanced filtering (e.g., Doctors with salaries higher than dentists).
  - Aggregate functions (e.g., Counting surgeries per doctor).
  - Joins (e.g., Linking patients with their medical records and doctors).
  - Data maintenance (Updates and Deletions based on specific conditions).
